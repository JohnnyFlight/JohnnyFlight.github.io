Pi Collisions

Introduction

I watched a video a while back by 3Blue1Brown about a curious way to calculate the digits of pi using the number of collisions between two masses and a wall.

I loved it, and thought it was really interesting, but thought little of it until I saw another video by The Coding Train where he attempted to recreate the simulation.

It worked, but he had to use increasingly larger time steps to be able to achieve greater accuracy.

He left the task of finding a better solution to the viewers, and I have decided to try my hand at one.

Instead of moving the blocks a little bit every frame and checking for a collision, I'm instead going to use linear algebra to determine when each collision is going to occur.

This reduces the computation significantly as you no longer have to test for thousands of collisions.

However, I will still fall foul of the limitations of 32-bit floating point values which will limit my accuracy.

I could find a Decimal library of some kind to solve this problem, but I'll probably just rewrite this in Python or C# as they already have Decimal types implemented.
